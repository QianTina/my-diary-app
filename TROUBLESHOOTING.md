# 故障排除指南

## 问题：Vite 报错 "does not provide an export named 'PaperStyle'"

这是一个常见的 Vite 热重载缓存问题。

### 解决方案

#### 方法 1：清理缓存并重启（推荐）

1. **停止开发服务器**（如果正在运行）
   - 在终端中按 `Ctrl+C`

2. **清理 Vite 缓存**
   ```bash
   rm -rf node_modules/.vite
   ```

3. **重启开发服务器**
   ```bash
   npm run dev
   ```

4. **硬刷新浏览器**
   - Mac: `Cmd + Shift + R`
   - Windows/Linux: `Ctrl + Shift + R`

#### 方法 2：完全清理（如果方法1不起作用）

1. **停止开发服务器**

2. **清理所有缓存**
   ```bash
   rm -rf node_modules/.vite
   rm -rf dist
   ```

3. **重新安装依赖**（可选，通常不需要）
   ```bash
   npm install
   ```

4. **重启开发服务器**
   ```bash
   npm run dev
   ```

5. **完全关闭并重新打开浏览器**

#### 方法 3：浏览器缓存清理

1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 如果问题仍然存在

1. 检查终端中的完整错误信息
2. 确认所有文件都已保存
3. 尝试重启 VS Code 或你的编辑器
4. 检查是否有其他 TypeScript 编译错误

### 常见问题

**Q: 为什么会出现这个错误？**  
A: Vite 的热模块替换（HMR）有时会缓存旧的模块信息，导致无法识别新添加的导出。

**Q: 这会影响生产构建吗？**  
A: 不会，这只是开发环境的问题。生产构建会正常工作。

**Q: 如何避免这个问题？**  
A: 在添加大量新文件后，建议重启开发服务器以确保所有模块正确加载。
