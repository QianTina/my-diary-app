# 使用指南

## 快速开始

### 1. 写第一篇日记
1. 在"写新日记"区域输入内容
2. （可选）添加标签：在标签输入框输入标签名，按回车添加
3. 点击"立即记录"保存

### 2. 使用 Markdown
日记支持 Markdown 格式，让你的内容更丰富：

```markdown
# 今天的心情

今天天气**很好**，心情*不错*！

## 做了什么
- 早上跑步
- 下午看书
- 晚上写代码

> 生活就是要充实！
```

点击"👁️ 预览"按钮查看渲染效果。

### 3. 标签管理

#### 添加标签
- 在编辑器下方的标签输入框输入标签名
- 按 `Enter` 键添加
- 可以添加多个标签，如：`开心`、`工作`、`旅行`

#### 删除标签
- 点击标签上的 `×` 按钮
- 或在输入框为空时按 `Backspace` 删除最后一个标签

#### 筛选日记
- 在搜索栏下方的标签云中点击标签
- 可以选择多个标签（AND 逻辑）
- 点击"清空筛选"取消所有标签筛选

### 4. 搜索日记
在搜索框输入关键词，实时过滤日记内容。

### 5. 编辑日记
1. 点击日记卡片右上角的"编辑"按钮
2. 修改内容和标签
3. 点击"确认更新"保存

### 6. 删除日记
点击"删除"按钮，确认后即可删除（不可恢复）。

### 7. 数据备份

#### 导出数据
1. 点击右上角"⚙️ 设置"
2. 选择导出格式：
   - **JSON**：完整数据备份，可用于导入恢复
   - **Markdown**：可读性强，便于分享或打印

#### 导入数据
1. 点击"设置" → "📤 从 JSON 导入"
2. 选择之前导出的 JSON 文件
3. 确认导入

⚠️ **注意**：导入会将数据添加到现有日记中，不会覆盖。

## 高级功能

### 草稿自动保存
- 输入的内容会自动保存为草稿
- 刷新页面后会自动恢复
- 提交或取消编辑后会清除草稿

### 数据存储模式

#### LocalStorage 模式（默认）
- 数据保存在浏览器本地
- 无需配置，开箱即用
- 隐私安全，数据不会上传

#### Supabase 模式（云端同步）
1. 创建 Supabase 项目
2. 执行 `sql/schema.sql` 创建表
3. 创建 `.env` 文件：
   ```env
   VITE_SUPABASE_URL=your_url
   VITE_SUPABASE_ANON_KEY=your_key
   ```
4. 重启应用，数据将自动同步到云端

## 快捷键

- `Enter`：在标签输入框中添加标签
- `Backspace`：在标签输入框为空时删除最后一个标签

## 常见问题

### Q: 数据会丢失吗？
A: LocalStorage 模式下，数据保存在浏览器中，除非清除浏览器数据，否则不会丢失。建议定期导出备份。

### Q: 可以在多个设备同步吗？
A: 使用 Supabase 模式可以实现云端同步。

### Q: 支持哪些 Markdown 语法？
A: 支持标题、粗体、斜体、列表、链接、代码、引用等常用语法。

### Q: 如何迁移数据？
A: 导出为 JSON 文件，在新设备上导入即可。

### Q: 标签有数量限制吗？
A: 没有硬性限制，但建议每条日记不超过 5 个标签，便于管理。

## 技巧与建议

1. **使用标签分类**：为日记添加有意义的标签，如 `#工作`、`#生活`、`#旅行`
2. **定期备份**：每月导出一次 JSON 备份
3. **善用 Markdown**：使用标题和列表让日记结构更清晰
4. **搜索技巧**：搜索支持中英文，可以搜索内容中的任何关键词
5. **标签命名**：使用简短、有意义的标签名，避免重复
